Import guide for Edge Favourites and step-by-step browser extension install.

🔖 Import from Edge

Export your Edge Favourites as an HTML file and import it here in one click.

  1. In Edge, click the ⋯ menu (top-right)
  2. Go to Favorites → Manage favorites
  3. Click the ⋯ inside Favorites → Export favorites
  4. Save the .html file anywhere on your device
  5. Return here and use the ☰ Import HTML menu action (in the top navbar)
  6. Select your exported file — duplicates are skipped automatically

Tip: chrome://favorites/all can also export from Chrome/Chromium.

Server-side CLI (large imports)

For very large collections run on the server:

python3 /root/xarta-node/blueprints-app/scripts/import_bookmarks.py \
  --file /tmp/favorites.html \
  --api http://127.0.0.1:8080

🧩 Browser Extension

Install the Blueprints Bookmarks extension in Edge for one-click save and inline search from any page.

  1. Use the ☰ Download extension menu action (in the top navbar) to download the extension zip
  2. Unzip it to a permanent folder on your device (e.g. Documents\blueprints-ext\)
  3. In Edge, navigate to edge://extensions
  4. Enable Developer mode (toggle, bottom-left)
  5. Click Load unpacked and select the unzipped folder
  6. Pin the extension (puzzle-piece icon → pin 📌)
  7. Click the extension icon → Options to set your API URL and secret

Options to configure in the extension:

API URL — detecting…
API Secret — your BLUEPRINTS_API_SECRET value from .env
Visit recorder — optional; auto-logs every page you visit

Chromium Manifest V3 — works in Edge and Chrome. Firefox not supported (different manifest).

⚠ Integrity FAILED

This node's database has been marked as degraded. Sync is paused — no outgoing writes will be sent to peers, and this node will not accept incoming sync actions until recovery.

Common cause: A Force Restore was applied (disaster recovery). The restored database is flagged as degraded until a healthy peer pushes its full backup to this node.

How to recover

  1. Wait — on every boot the app requests a full backup from the first reachable peer. If peers are online, recovery is automatic within ~10–30 s of (re)starting the service.
  2. Restart the service — if the node has been running for a while without recovering, restarting triggers boot-catchup again: systemctl restart blueprints-app
  3. Manual restore — navigate to Settings › Backups and restore from a known-good backup, then restart the service.
  4. No peers available? — if this is the only node, you can reset integrity manually from the SQLite shell: UPDATE sync_meta SET value='true' WHERE key='integrity_ok'; Only do this if you are certain the data is uncorrupted.
